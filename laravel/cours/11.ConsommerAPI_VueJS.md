# Chapitre 11 – Consommer une API REST avec Vue.js 3

## 11.1 Objectif général du chapitre

Au chapitre 10, tu as construit une **API REST complète** avec Laravel et Sanctum.

Ce chapitre te montre comment un **front-end Vue.js 3** peut consommer cette API :

* afficher la liste des services (requête GET sans token)
* permettre à un utilisateur de se connecter (POST → récupérer un token)
* stocker le token en `localStorage`
* envoyer ce token automatiquement dans chaque requête protégée
* afficher, créer et annuler des réservations

> **Architecture** : deux serveurs séparés qui communiquent via HTTP.
> Laravel (port 8000) ← API → Vue.js (port 5173)

---

## 11.5 Étape 1 – Configurer le CORS dans Laravel

### Pourquoi le CORS ?

Quand Vue.js (port 5173) appelle Laravel (port 8000), le navigateur bloque la requête
par défaut car les deux origines sont différentes.

Il faut explicitement **autoriser** l'origine du front-end dans Laravel.

### Fichier complet : `config/cors.php`

```php
<?php
return [
    'paths'            => ['api/*'],
    'allowed_methods'  => ['*'],
    'allowed_origins'  => [
        'http://localhost:5173',
        'http://127.0.0.1:5173',
    ],
    'allowed_headers'  => ['*'],
    'exposed_headers'  => [],
    'max_age'          => 0,
    'supports_credentials' => false,
];
```

---

### Enregistrer le middleware CORS dans Laravel 11

📁 `bootstrap/app.php`

```php
use Illuminate\Http\Middleware\HandleCors;  // ← ajouter cet import

->withMiddleware(function (Middleware $middleware): void {
    $middleware->prepend(HandleCors::class); // ← ajouter cette ligne
    $middleware->alias([
        'role' => RoleMiddleware::class,
    ]);
})
```

### Vider le cache

```bash
php artisan config:clear
```

> Désormais, les requêtes venant de `http://localhost:5173` sont acceptées.

## 11.7 Étape 3 – Configurer Vite (proxy vers Laravel)

### Pourquoi un proxy ?

Au lieu de configurer le CORS de deux côtés,
Vite peut **proxy** les requêtes `/api` vers Laravel en interne.

Avantage : **aucune erreur CORS** en développement.

### Fichier complet : `front-vue/vite.config.js`

```js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    port: 5173,
    proxy: {
      // Toute requête commençant par /api est redirigée vers Laravel
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      }
    }
  }
})
```

### Résultat

```
Vue (5173) → /api/v1/services → Vite proxy → Laravel (8000)/api/v1/services
```

## 11.9 Étape 5 – Le service API (Axios)

Ce fichier centralise toute la communication avec le back-end.

### Fichier complet : `front-vue/src/services/api.js`

```js
import axios from 'axios'
import router from '../router'

const api = axios.create({
  baseURL: '/api/v1',            // proxy Vite → http://localhost:8000/api/v1
  headers: {
    'Accept': 'application/json',
    'Content-Type': 'application/json',
  },
})

// Intercepteur REQUÊTE : ajouter le token Bearer automatiquement
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// Intercepteur RÉPONSE : si token expiré (401) → rediriger vers /login
api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('token')
      localStorage.removeItem('user')
      router.push('/login')
    }
    return Promise.reject(error)
  }
)

export default api
```

---

### Comment utiliser api.js dans un composant

```js
import api from '../services/api'

// GET (sans token)
const { data } = await api.get('/services')

// POST (avec token — ajouté automatiquement)
const { data } = await api.post('/reservations', {
  service_id: 1,
  date_reservation: '2026-05-10',
  heure_reservation: '09:30',
})
```

> Le token est automatiquement ajouté par l'intercepteur.
> Tu n'as **jamais** besoin de l'écrire manuellement dans le composant.

## 11.10 Étape 6 – Gérer l'authentification (composable)

Un **composable** Vue est une fonction réutilisable qui encapsule de la logique.

### Fichier complet : `front-vue/src/composables/useAuth.js`

```js
import { reactive, computed } from 'vue'
import api from '../services/api'

// État global partagé entre tous les composants
const state = reactive({
  user:  JSON.parse(localStorage.getItem('user'))  || null,
  token: localStorage.getItem('token') || null,
})

export function useAuth() {
  const isLoggedIn = computed(() => !!state.token)

  async function login(email, password) {
    const { data } = await api.post('/login', { email, password })
    _sauvegarder(data)
    return data
  }

  async function register(name, email, password, password_confirmation) {
    const { data } = await api.post('/register', {
      name, email, password, password_confirmation,
    })
    _sauvegarder(data)
    return data
  }

  async function logout() {
    try { await api.post('/logout') } catch (_) {}
    finally { _effacer() }
  }
```

---

```js
  // fonctions privées
  function _sauvegarder(data) {
    state.token = data.token
    state.user  = data.user
    localStorage.setItem('token', data.token)
    localStorage.setItem('user',  JSON.stringify(data.user))
  }

  function _effacer() {
    state.token = null
    state.user  = null
    localStorage.removeItem('token')
    localStorage.removeItem('user')
  }

  return { user: state, isLoggedIn, login, register, logout }
}
```

### Utilisation dans n'importe quel composant

```js
import { useAuth } from '../composables/useAuth'

const { user, isLoggedIn, login, logout } = useAuth()

// Connexion
await login('jean@exemple.com', 'password')

// Vérifier si connecté
if (isLoggedIn.value) { ... }

// Accéder au rôle
console.log(user.user?.role)   // 'patient' | 'medecin' | 'admin'
```

## 11.11 Étape 7 – Le routeur Vue (navigation protégée)

### Fichier complet : `front-vue/src/router/index.js`

```js
import { createRouter, createWebHistory } from 'vue-router'

// Lazy loading : chaque vue n'est chargée que quand on y navigue
const HomeView          = () => import('../views/HomeView.vue')
const LoginView         = () => import('../views/LoginView.vue')
const RegisterView      = () => import('../views/RegisterView.vue')
const ServicesView      = () => import('../views/ServicesView.vue')
const ServiceDetail     = () => import('../views/ServiceDetail.vue')
const ReservationCreate = () => import('../views/ReservationCreate.vue')
const MesReservations   = () => import('../views/MesReservations.vue')

const routes = [
  { path: '/',               name: 'home',             component: HomeView },
  { path: '/login',          name: 'login',            component: LoginView,
    meta: { guestOnly: true } },
  { path: '/register',       name: 'register',         component: RegisterView,
    meta: { guestOnly: true } },
  { path: '/services',       name: 'services',         component: ServicesView },
  { path: '/services/:id',   name: 'service-detail',   component: ServiceDetail },
  { path: '/reserver/:id',   name: 'reserver',         component: ReservationCreate,
    meta: { requiresAuth: true, role: 'patient' } },
  { path: '/mes-reservations', name: 'mes-reservations', component: MesReservations,
    meta: { requiresAuth: true, role: 'patient' } },
]
```

---

### Navigation guard (protection des routes)

```js
const router = createRouter({
  history: createWebHistory(),
  routes,
  scrollBehavior: () => ({ top: 0 }),
})

router.beforeEach((to) => {
  const token = localStorage.getItem('token')
  const user  = JSON.parse(localStorage.getItem('user') || 'null')

  // Route réservée aux invités : si connecté → rediriger
  if (to.meta.guestOnly && token) {
    return { name: 'services' }
  }

  // Route protégée : si non connecté → aller au login
  if (to.meta.requiresAuth && !token) {
    return { name: 'login', query: { redirect: to.fullPath } }
  }

  // Restriction de rôle
  if (to.meta.role && user?.role !== to.meta.role) {
    return { name: 'home' }
  }
})

export default router
```

> `meta.requiresAuth: true` → la route est privée (token obligatoire)
> `meta.guestOnly: true` → la route n'est accessible qu'aux non-connectés

## 11.13 Étape 9 – Vue Login

### Logique complète de `LoginView.vue`

```vue
<script setup>
import { ref, inject } from 'vue'
import { useRouter }   from 'vue-router'
import { useAuth }     from '../composables/useAuth'

const router      = useRouter()
const { login }   = useAuth()
const showMessage = inject('showMessage')   // message flash depuis App.vue

const form = ref({ email: '', password: '' })
const erreur     = ref(null)
const chargement = ref(false)

async function handleLogin() {
  erreur.value     = null
  chargement.value = true
  try {
    const data = await login(form.value.email, form.value.password)

    showMessage(`Bienvenue, ${data.user.name} !`)

    // Redirection selon le rôle
    if (data.user.role === 'patient') router.push('/mes-reservations')
    else                              router.push('/services')

  } catch (e) {
    erreur.value = e.response?.data?.message || 'Identifiants incorrects.'
  } finally {
    chargement.value = false
  }
}
</script>
```

---

### Template du formulaire de connexion

```vue
<template>
  <form @submit.prevent="handleLogin">
    <!-- Erreur -->
    <div v-if="erreur" class="alert alert-danger">{{ erreur }}</div>

    <!-- Email -->
    <div class="mb-3">
      <label class="form-label">Email</label>
      <input v-model="form.email" type="email" class="form-control" required />
    </div>

    <!-- Mot de passe -->
    <div class="mb-4">
      <label class="form-label">Mot de passe</label>
      <input v-model="form.password" type="password" class="form-control" required />
    </div>

    <!-- Bouton avec état de chargement -->
    <button type="submit" class="btn btn-primary w-100" :disabled="chargement">
```

## 11.15 Étape 11 – Vue Réservation (appel POST protégé)

### Logique : `ReservationCreate.vue`

```vue
<script setup>
import { ref, onMounted, inject } from 'vue'
import { useRoute, useRouter }    from 'vue-router'
import api from '../services/api'

const route       = useRoute()
const router      = useRouter()
const showMessage = inject('showMessage')

const service = ref(null)
const form    = ref({
  date_reservation:  '',
  heure_reservation: '',
  commentaire:       '',
})
const erreurs = ref(null)

// 1. Charger le service depuis l'API
onMounted(async () => {
  const { data } = await api.get(`/services/${route.params.id}`)
  service.value = data.data ?? data
})

// 2. Envoyer la réservation
async function handleSubmit() {
  erreurs.value = null
  try {
    await api.post('/reservations', {
      service_id:        route.params.id,
      ...form.value,
    })
    showMessage('Réservation confirmée !')
    router.push('/mes-reservations')
  } catch (e) {
    if (e.response?.status === 422) erreurs.value = e.response.data.errors
  }
}
</script>
```

---

### Ce qui se passe en coulisses

```
1. Utilisateur remplit le formulaire → soumet
                    ↓
2. Vue appelle : api.post('/reservations', { ... })
                    ↓
3. Axios intercepte → ajoute automatiquement le header :
   Authorization: Bearer 1|abc123xyz...
                    ↓
4. Laravel reçoit la requête :
   - vérifie le token Sanctum
   - vérifie le rôle (middleware 'role:patient')
   - crée la réservation en base
   - retourne { message, data }
                    ↓
5. Vue reçoit la réponse → affiche le message → redirige
```

## 11.16 Étape 12 – Vue Mes Réservations + Annulation

### Charger les réservations du patient connecté

```vue
<script setup>
import { ref, onMounted, inject } from 'vue'
import api from '../services/api'

const showMessage  = inject('showMessage')
const reservations = ref([])

// GET /api/v1/reservations
// Le token est envoyé automatiquement → Laravel filtre par user_id
onMounted(async () => {
  const { data } = await api.get('/reservations')
  reservations.value = data.data ?? data
})

// Annuler une réservation
async function annuler(id) {
  if (!confirm('Annuler cette réservation ?')) return
  try {
    await api.post(`/reservations/${id}/cancel`)
    showMessage('Réservation annulée.')
    // Recharger la liste
    const { data } = await api.get('/reservations')
    reservations.value = data.data ?? data
  } catch (e) {
    showMessage(e.response?.data?.message || 'Erreur.', 'danger')
  }
}
</script>
```

---

### Template de la liste

```vue
<template>
  <div class="card shadow">
    <div class="card-body p-0">
      <table class="table table-hover mb-0">
        <thead class="table-light">
          <tr>
            <th>Service</th><th>Date</th><th>Statut</th><th>Action</th>
          </tr>
        </thead>
        <tbody>
          <tr v-for="r in reservations" :key="r.id">
            <td>{{ r.service?.titre }}</td>
            <td>{{ new Date(r.date_reservation).toLocaleDateString('fr-FR') }}</td>
            <td>
              <!-- Statut coloré -->
              <span :class="`badge bg-${r.statut === 'confirmee' ? 'success'
                    : r.statut === 'annulee' ? 'danger'
                    : r.statut === 'effectuee' ? 'info' : 'warning'}`">
                {{ r.statut }}
              </span>
            </td>
            <td>
              <!-- Bouton annuler (seulement si en_attente) -->
              <button v-if="r.statut === 'en_attente'"
                      class="btn btn-outline-danger btn-sm"
                      @click="annuler(r.id)">
                Annuler
              </button>
            </td>
          </tr>
        </tbody>
      </table>
    </div>
  </div>
</template>
```

## 11.21 Erreurs fréquentes et solutions

### Erreur CORS

```
Access to XMLHttpRequest at 'http://localhost:8000/api/v1/...'
from origin 'http://localhost:5173' has been blocked by CORS policy
```

**Solution** : vérifier `config/cors.php` et que `HandleCors::class` est dans `bootstrap/app.php`.

---

### Erreur 401 Unauthorized

```json
{ "message": "Unauthenticated." }
```

**Cause** : le token n'est pas envoyé.
**Solution** : vérifier que `localStorage.getItem('token')` retourne bien le token.
Vérifier l'intercepteur dans `api.js`.

---

### Erreur 403 Forbidden

```json
{ "message": "Accès refusé." }
```

**Cause** : le rôle de l'utilisateur ne correspond pas.
**Solution** : vous avez utilisé un compte médecin pour une route patient.

---

### Données non affichées (undefined)

```js
// ❌ Mauvais
services.value = data   // si l'API retourne { data: [...] }

// ✅ Correct
services.value = data.data ?? data  // compatible les deux formats
```
