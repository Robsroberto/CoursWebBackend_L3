## 2.1 `.env` — Paramètres MySQL

```
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=reservation_app
DB_USERNAME=root
DB_PASSWORD=
```

### Ce que Laravel fait réellement :

* Il lit `.env` à chaque lancement (sauf si config en cache).
* Il prépare une connexion PDO avec ces paramètres.
* Il utilise `DB_CONNECTION` pour choisir quel driver utiliser (`mysql`, `sqlite`, `pgsql`...).
* Toutes les migrations, seeders et Eloquent ORM passeront par cette connexion.

### Pièges fréquents des étudiants :

1. Ils laissent `DB_CONNECTION=sqlite`.
2. Ils oublient de créer la base dans phpMyAdmin.
3. Ils mettent un mot de passe alors que XAMPP n’en a pas.
4. Ils oublient de supprimer `DB_DATABASE=/path/to/database.sqlite`.

---

## 2.2 `php.ini` — Activation de pdo_mysql

Ajouter :

```
extension=pdo_mysql
```

### Pourquoi ?

Laravel utilise PDO comme couche d’abstraction.
Sans cette extension :

* impossible de communiquer avec MySQL,
* erreur → *“could not find driver”*

### Pièges fréquents :

* L’étudiant modifie le mauvais php.ini.
* Il ne redémarre pas Apache.
* Il a plusieurs versions de PHP dans son PATH Windows.

---

## 2.3 Migration `add_role_to_users_table`

```php
$table->enum('role', ['admin', 'medecin', 'patient'])->default('patient');
```

### Explication interne :

* `enum` limite les valeurs possibles.
* Toutes les futures vérifications “security / redirect / middleware” utiliseront ce champ.
* Si un étudiant oublie cette migration → impossible d’implémenter les rôles.

---

## 2.4 Migration `create_services_table`

### Points importants :

* `decimal(8,2)` → utilisé pour le prix (ex: 15000.00).
* `foreign('medecin_id')` → assure que le médecin doit exister dans `users`.
* `nullable()` → permet services sans médecin attribué.

### Erreurs fréquentes :

1. L’étudiant supprime accidentellement la clé étrangère.
2. Il utilise un “integer” simple qui ne correspond pas au type de clé étrangère.
3. Il oublie MySQL → erreur : *“Foreign key constraint is incorrectly formed”*

---

## 2.5 Migration `create_reservations_table`

### Points de vigilance :

* `user_id` = patient
* `service_id` = service réservé
* `enum` pour le statut
* date + time (2 colonnes séparées)

### Laravel ORM :

* Appellera automatiquement les relations `user()` et `service()`
* Associera la clé étrangère avec un modèle Eloquent

---

## 2.6 Seeder `DemoSeeder`

Il crée :

* un admin
* un médecin
* un service exemple

### Pourquoi un seeder ?

* les étudiants ont souvent besoin de comptes pré-configurés
* il accélère les tests et évite les aller-retours manuels
* permet d'avoir un médecin pour réaliser des tests de relation

### Erreurs fréquentes :

* Ils oublient d’exécuter les seeders.
* Ils insèrent des données avant les migrations → erreur.
* Ils créent une migration dans le futur → migration désynchronisée.

# 4. GUIDE DE DEBUGGING : ERREURS POSSIBLES ET SOLUTIONS

Voici **toutes les erreurs fréquentes** que tes étudiants rencontreront + **la solution**.

---

### ❌ Erreur 1 : “could not find driver”

**Cause probable :** pdo_mysql non activé.

**Solution :**

1. Modifier php.ini
2. Activer `extension=pdo_mysql`
3. Redémarrer Apache
4. Faire `php --ini` pour vérifier le bon php.ini

---

### ❌ Erreur 2 : “SQLSTATE[HY000] [1049] Unknown database”

**Cause :** la base `reservation_app` n’existe pas.

**Solution :** créer la base dans phpMyAdmin.

---

### ❌ Erreur 3 : “Access denied for user 'root'@'localhost'”

**Cause :** mot de passe erroné.

**Solution :** sous XAMPP → root n’a pas de mot de passe.

---

### ❌ Erreur 4 : “Foreign key constraint incorrectly formed”

Causes possibles :

* type de colonne incorrect (int vs bigint)
* cascade manquante
* table users non migrée

**Solution :**
Mettre `unsignedBigInteger` dans toutes les clés étrangères.

---

### ❌ Erreur 5 : “Base table or view already exists”

**Cause :** l’étudiant a exécuté plusieurs fois les migrations.

**Solution :**

```bash
php artisan migrate:fresh
```

---

### ❌ Erreur 6 : “Class not found” pour le seeder

**Cause :** oubli d’ajouter :

```php
use App\Models\User;
use App\Models\Service;
```

**Solution :** ajouter les bons imports.

---

### ❌ Erreur 7 : Migrations exécutées dans SQLite par erreur

**Cause :** `.env` pas mis à jour.

**Symptôme :** table `services` introuvable dans MySQL.

**Solution :**

```bash
php artisan config:clear
php artisan cache:clear
php artisan migrate:fresh
```
