Le semestre sera évalué à travers **des projets pratiques obligatoires sur GitHub** et un **examen final individuel** basé sur une version améliorée du projet fil rouge.

---

#   1. Organisation générale

Votre note du semestre sera composée de :

1. **Contrôle continu (50%)**
   → basé sur **5 projets pratiques**, tous réalisés sur GitHub.
2. **Examen final individuel (50%)**

⚠️ **GitHub est obligatoire pour chaque projet.**
⚠️ **Aucun commit personnel = 0**, même si le groupe rend un projet complet.

---

#   2. Projets du contrôle continu (5 projets)

Chaque projet est noté **/20**.

1. **TP1 : Projet POO & MVC PHP** (CRUD + validation + catégories)
2. **TP2 : —** *(sera annoncé en cours)*
3. **TP3 : —** *(sera annoncé en cours)*
4. **TP4 : —** *(sera annoncé en cours)*
5. **TP5 : —** *(sera annoncé en cours)*

👉 **La note de contrôle continu = moyenne des 5 projets**.

---

#   3. Contrainte GitHub (obligatoire et non négociable)

Pour chaque projet :

* Vous devez créer un **repository GitHub**
* Vous devez ajouter le professeur en collaborateur
* Chaque membre doit faire des **commits personnels et visibles**
* Le fichier **README** doit mentionner ce que chacun a fait

⚠️ **Sans commits personnels → note = 0 sur le projet**
⚠️ **Pas de fork, pas de ZIP, pas de transfert WhatsApp**

---

#   4. Grille de notation pour chaque projet (sur 20)

###   1. Participation individuelle (6 pts)

* commits personnels : 2 pts
* commits utiles (pas vides) : 2 pts
* rôle clair dans le README : 2 pts

###   2. Qualité technique du code (8 pts)

* consignes respectées : 3 pts
* bonne organisation (MVC / fichiers / clarté) : 2 pts
* absence d’erreurs majeures : 2 pts
* code lisible : 1 pt

###   3. Fonctionnalités demandées (4 pts)

* fonctionnalités obligatoires opérationnelles : 3 pts
* bonus / améliorations : 1 pt

---

#   5. Examen final individuel (50%)

L’examen final sera un **projet individuel obligatoire** :

Vous devrez :

* travailler **seul**,
* rendre le projet **sur GitHub**,
* proposer une version **professionnelle du projet fil rouge**.

⚠️ Projet trop ressemblant à celui d’un camarade → **note annulée**.

---

#   6. Résumé

* GitHub obligatoire
* 5 projets = 50% de la note
* Examen final = 50%
* Pas de commit personnel = **0**
* Chaque projet sera annoncé au fur et à mesure
* Objectif : construire progressivement une vraie compétence backend
