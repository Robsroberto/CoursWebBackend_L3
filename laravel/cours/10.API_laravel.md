## 10.1 Rappel clair : API et REST

### 10.1.1 API (Application Programming Interface)

Une API est un moyen standard d’accéder aux fonctionnalités de ton backend **sans interface HTML**.

* Application web : le backend renvoie des pages (HTML)
* API : le backend renvoie des données (souvent JSON)

### 10.1.2 REST (principes simples)

REST impose une manière cohérente de nommer et d’utiliser les URL :

* Une URL représente une **ressource** (un objet métier)
* On utilise des verbes HTTP pour manipuler cette ressource

Exemples de ressources du projet fil rouge :

* services
* reservations
* medecins (users dont role = medecin)

Exemples de verbes HTTP :

* GET : lire
* POST : créer
* PUT/PATCH : modifier
* DELETE : supprimer

## 10.2 Où écrire les routes API dans Laravel ?

Laravel sépare volontairement :

* `routes/web.php` : routes web (HTML, sessions, vues Blade)
* `routes/api.php` : routes API (JSON)

Important :

* toutes les routes définies dans `api.php` sont automatiquement préfixées par `/api`

Donc :

```php
Route::get('/ping', ...)
```

dans `api.php` donne :

```
GET /api/ping
```

## 10.4 Étape 2 – Organiser l’API proprement (versionning v1)

Pourquoi versionner ?
Parce que dans une vraie application, l’API évolue.
Tu veux pouvoir ajouter `/api/v2/...` plus tard sans casser les clients existants.

### Modification : `routes/api.php` (fichier complet)

```php
<?php

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\Api\V1\ServiceApiController;

Route::get('/ping', function () {
    return response()->json([
        'success' => true,
        'message' => 'API OK'
    ], 200);
});

Route::prefix('v1')->group(function () {
    Route::get('/services', [ServiceApiController::class, 'index']);
    Route::get('/services/{service}', [ServiceApiController::class, 'show']);
});
```

### Explication

* `prefix('v1')` : ajoute automatiquement `/v1` devant toutes les routes du groupe
* donc les endpoints deviennent :

  * `GET /api/v1/services`
  * `GET /api/v1/services/{service}`

## 10.6 Étape 4 – Implémenter le contrôleur API (code complet)

### Fichier complet : `app/Http/Controllers/Api/V1/ServiceApiController.php`

Copie-colle ce contenu :

```php
<?php

namespace App\Http\Controllers\Api\V1;

use App\Http\Controllers\Controller;
use App\Models\Service;
use Illuminate\Http\Request;

class ServiceApiController extends Controller
{
    /**
     * GET /api/v1/services
     * Retourne la liste des services en JSON.
     */
    public function index(Request $request)
    {
        // On prépare une requête Eloquent (sans l'exécuter tout de suite)
        $query = Service::with('medecin')->orderBy('id', 'desc');

        // Filtrage optionnel : ?search=mot
        if ($request->filled('search')) {
            $query->where('titre', 'like', '%' . $request->search . '%');
        }

        // Pagination (10 par page)
        $services = $query->paginate(10);

        // Réponse JSON structurée
        return response()->json([
            'success' => true,
            'data' => $services
        ], 200);
    }

    /**
     * GET /api/v1/services/{service}
     * Retourne le détail d’un service en JSON.
     */
    public function show(Service $service)
    {
        // Charger la relation médecin (si pas déjà chargée)
        $service->load('medecin');

        return response()->json([
            'success' => true,
            'data' => $service
        ], 200);
    }
}
```

### Explications importantes

#### 1) `Service::with('medecin')`

* évite le problème N+1
* récupère les services + les médecins associés en une logique optimisée

#### 2) `paginate(10)`

* renvoie une structure JSON avec :

  * data (la liste)
  * meta (infos pagination)
  * links (liens pages)
* indispensable pour une API professionnelle

#### 3) `Route Model Binding`

Dans `show(Service $service)` :

* Laravel récupère automatiquement le service correspondant à l’ID

## 10.9 Étape 7 – Utiliser une Resource Laravel (contrôle du JSON)

### Pourquoi ?

Retourner le modèle brut peut :

* exposer des champs inutiles,
* varier selon les relations,
* rendre ton API instable.

Les Resources permettent de définir exactement le JSON de sortie.

---

### Commande

```bash
php artisan make:resource ServiceResource
```

### Ce que la commande crée

Un fichier :

```
app/Http/Resources/ServiceResource.php
```

---

### Fichier complet : `app/Http/Resources/ServiceResource.php`

```php
<?php

namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class ServiceResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'titre' => $this->titre,
            'description' => $this->description,
            'prix' => $this->prix,
            'duree' => $this->duree,
            'statut' => $this->statut,
            'medecin' => $this->whenLoaded('medecin', function () {
                return [
                    'id' => $this->medecin->id,
                    'name' => $this->medecin->name,
                    'email' => $this->medecin->email,
                ];
            }),
        ];
    }
}
```

---

### Adapter le contrôleur API (fichier complet)

On modifie `ServiceApiController` pour utiliser la Resource.

```php
<?php

namespace App\Http\Controllers\Api\V1;

use App\Http\Controllers\Controller;
use App\Http\Resources\ServiceResource;
use App\Models\Service;
use Illuminate\Http\Request;

class ServiceApiController extends Controller
{
    public function index(Request $request)
    {
        $query = Service::with('medecin')->orderBy('id', 'desc');

        if ($request->filled('search')) {
            $query->where('titre', 'like', '%' . $request->search . '%');
        }

        $services = $query->paginate(10);

        return response()->json([
            'success' => true,
            'data' => ServiceResource::collection($services)
        ], 200);
    }

    public function show(Service $service)
    {
        $service->load('medecin');

        return response()->json([
            'success' => true,
            'data' => new ServiceResource($service)
        ], 200);
    }
}
```

# 10.1 Rappel : Qu’est-ce qu’une API REST ?

Une API REST est une interface qui permet :

* d’exposer des données
* d’exposer des traitements
* en utilisant HTTP
* en retournant généralement du JSON

Une API REST ne renvoie PAS de Blade, PAS de HTML.
Elle renvoie des données structurées.

Exemple :

GET /api/services

Réponse :

```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "titre": "Consultation générale",
      "prix": 15000
    }
  ]
}
```

# 10.10 Étape 7 – Créer AuthApiController

Commande :

```bash
php artisan make:controller Api/V1/AuthApiController
```

---

## Controller complet

```php
<?php

namespace App\Http\Controllers\Api\V1;

use App\Http\Controllers\Controller;
use App\Models\User;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Hash;

class AuthApiController extends Controller
{
    public function login(Request $request)
    {
        $request->validate([
            'email' => 'required|email',
            'password' => 'required'
        ]);

        $user = User::where('email', $request->email)->first();

        if (!$user || !Hash::check($request->password, $user->password)) {
            return response()->json([
                'success' => false,
                'message' => 'Identifiants incorrects'
            ], 401);
        }

        $token = $user->createToken('api-token')->plainTextToken;

        return response()->json([
            'success' => true,
            'token' => $token
        ]);
    }
}
```

# 10.12 Étape 9 – Ajouter Middleware Role (Laravel 12)

Commande :

```bash
php artisan make:middleware RoleMiddleware
```

---

## Code complet

```php
<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;

class RoleMiddleware
{
    public function handle(Request $request, Closure $next, string $role)
    {
        $user = $request->user();

        if ($user->role !== $role) {
            return response()->json([
                'success' => false,
                'message' => 'Accès interdit'
            ], 403);
        }

        return $next($request);
    }
}
```

---

## Enregistrer dans bootstrap/app.php

```php
->withMiddleware(function ($middleware) {
    $middleware->alias([
        'role' => \App\Http\Middleware\RoleMiddleware::class,
    ]);
})
```

# 10.19 Implémenter le contrôleur Réservations

Code complet :

```php
<?php

namespace App\Http\Controllers\Api\V1;

use App\Http\Controllers\Controller;
use App\Models\Reservation;
use Illuminate\Http\Request;

class ReservationApiController extends Controller
{
    public function index()
    {
        $reservations = Reservation::with(['service', 'patient'])
            ->get();

        return response()->json([
            'success' => true,
            'data' => $reservations
        ], 200);
    }

    public function show(Reservation $reservation)
    {
        $reservation->load(['service', 'patient']);

        return response()->json([
            'success' => true,
            'data' => $reservation
        ], 200);
    }

    public function store(Request $request)
    {
        $validated = $request->validate([
            'service_id' => 'required|exists:services,id',
            'patient_id' => 'required|exists:users,id',
            'date_reservation' => 'required|date'
        ]);

        $reservation = Reservation::create($validated);

        return response()->json([
            'success' => true,
            'data' => $reservation
        ], 201);
    }

    public function update(Request $request, Reservation $reservation)
    {
        $validated = $request->validate([
            'date_reservation' => 'required|date'
        ]);

        $reservation->update($validated);

        return response()->json([
            'success' => true,
            'data' => $reservation
        ]);
    }

    public function destroy(Reservation $reservation)
    {
        $reservation->delete();

        return response()->json([
            'success' => true,
            'message' => 'Réservation supprimée'
        ]);
    }
}
```

# 10.22 Filtrer les services par médecin

Ce que l’on fait :
Permettre de filtrer par medecin_id via URL.

Modifier index() de ServiceApiController :

```php
public function index(Request $request)
{
    $query = Service::with('medecin');

    if ($request->has('medecin_id')) {
        $query->where('medecin_id', $request->medecin_id);
    }

    $services = $query->paginate(5);

    return response()->json($services);
}
```
